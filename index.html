Phase 0 · Triage

Before you forge, you route.

Every input is evaluated against your existing skill library. SkillForge picks the cheapest path that still works — reusing, improving, composing, or creating only when necessary.

The 4-Phase Architecture

Analyze. Specify. Generate. Approve.

A single skill traverses four autonomous phases before it is allowed out of the forge. Each phase has its own inputs, outputs, and quality gates.

§ 02 / 05
PHASE · 0
Triage
Classify the request. Route to re-use, improvement, composition, or new creation.
OUT → route_decision
PHASE · 1
Deep Analysis
11 thinking lenses pass over the problem. Regression loop interrogates every assumption.
OUT → analysis_bundle
PHASE · 2
Specification
Insights codified into an XML spec. Degrees of freedom set per design decision.
OUT → skill_spec.xml
PHASE · 3
Generate + Iterate
Fresh context builds the skill from spec, then refines against the spec before review.
OUT → skill_candidate
PHASE · 4
Synthesis Panel
Design, Audience, Evolution, Script agents. Unanimous approval or back to iterate.
OUT → skill_approved

Phase 4 · Multi-agent Synthesis

A panel of experts. Unanimous or back to the fire.

Before a skill ships, four specialized agents independently score it. One dissent sends it back through iteration. Evolution must score at least 7/10 or the skill is rejected.

§ 04 / 05
9/10

Design Agent

Structure, naming, frontmatter, file layout. Checks that the skill follows the forge's patterns and is mechanically sound.

APPROVEDEVAL · DESIGN
8/10

Audience Agent

Clarity, discoverability, completeness. Will the model actually load this skill at the right moment, and succeed once it does?

APPROVEDEVAL · AUDIENCE
7/10

Evolution Agent

Timelessness and extensibility. Will this skill still be correct after the next model release? Minimum score 7/10.

APPROVEDEVAL · EVOLUTION
9/10

Script Agent

Conditional. When scripts are present, validates code quality, safety, and degrees-of-freedom fit.

APPROVEDEVAL · SCRIPTS
Core doctrine

Three principles. No exceptions.

§ 05 / 05
PRINCIPLE I

Engineer for agents.

Context windows are a public good. Every line in SKILL.md competes with the user's actual work. Slim cores, lazy-loaded references, simplified frontmatter.

  • SKILL.md < 500 lines
  • name + description frontmatter
  • Automated validation
PRINCIPLE II

Systematize rigor.

Quality is not a phase — it is the substrate. Regression questioning, eleven lenses, multi-agent review, degrees-of-freedom assessment at every decision point.

  • 4-phase architecture
  • Regression loop
  • Unanimous synthesis
PRINCIPLE III

Design for evolution.

A skill that works once is an artifact. A skill that keeps working is infrastructure. Evolution scoring is a hard gate, not a nice-to-have.

  • Dedicated Evolution agent
  • Timelessness ≥ 7/10
  • Iteration as a formal step
